#  Todo 도메인 모델

## 요구사항
- "회원"의 정보를 변경하는 기능을 제공한다.
  * password
  * email
  * 사진 ( 증명사진 )
  * 주소
- "관리자"는 회원 상태를 "활성화" , "비활성화" 할수 있다
- "할일"의 관리 기능이란 "할일"작성 및 삭제  , 편집 등의 기능을 제공한다.
- "할일"을 category 별로 보여줄수 있어야 한다.
- "할일"을 진행율( %) 기준으로 보여주는 기능을 제공해야 된다.
- "할일"의 현재 진행상태 ( 진행중 , 완료 , 기타 ... )를 보여줘야 한다.
- "할일"의 진행상태가 변경시마다 email을 발송한다.
- "할일"의 시작및 완료기간을 명시한다.
- "할일"은 "시작"을 해야 "진행중"으로 상태값이 변경된다.
- "할일"은 시작일자가 지나도 "시작"을 하지 않을경우 "지연"상태로 변경된다.
- "할일"이 완료가 되지 않았을경우 완료날짜 하루전에 이메일을 보낸다.
- "할일"은 100분율 단위로 퍼센트를 설정 및 노출한다.
- "할일"은 우선순위를 기준으로 진행 할수 있다.
- 우선순위 변경시 email이 발송된다. ( before -> after )
- "할일"은 완료기간을 연장가능하다. 단 이경우에는 "사유"를 작성해야 된다.
- "할일"이 완료되지 않고 완료기간이 지날경우 "만료"상태값으로 변경
- "할일"은 "카테고리" 별로 분류 가능하다.
- "할일"목록에서  클릭시 선택된 "할일"의 상세보기로 이동한다.
- "할일"목록에서 선택한 할일을 삭제 , 편집 가능하다.
- "삭제" 선택시에는 "삭제 질의 팝업"으로 확인 받아야 한다.

### [ 회원 애크리게이트 ] 
### 회원 (Member)
_Aggregate Root_
#### 속성
- `id`: 모델의 고유한값
- `email`: 이메일 ( Email ) 
- `userid`: signin 아이디 ( 등록후 변경할수 없다 )
- `password`:  암호 ( Password )
- `status` :  회원의 현재 상태 ( MemberStatus )
- `detail` : MemberDetail 1:1

#### 행위
- `static create()`: 회원 등록. 회원상태 (status)는 비활성 상태로 등록한다.
- `activate()`: 회원 상태를 활성상태로 변경한다.
- `withdraw()`: 회원 상태를 탈퇴 상태로 변경한다.
- `updateInfo()`: 회원 정보를 변경한다.
  * 경우에 따라 detail의 changeProfileImage() , changeAddress() 를 호출한다.
- `isActivate()`: 현재 활성화 상태인지 반환

#### 규칙  
- '회원활성화'는 회원의 상태가 '비활성화(deactivate)' 상태일 경우에만 활성화 가능하다.
- '회원탈퇴'는 회원의 상태가 '활성화 ( activate )' 상태일경우에만 탈퇴 가능하다.
- 'email'은 email 형식에 맞아야만 의미가 있다
- 'password'는 4 ~ 8 자리의 길이만 허용한다.


### 회원상세 ( MemberDetail )
_Entity_
#### 속성
- `id`:고유한 값
- `address`: 주소
- `profileImage`: 증명사진
- `registeredAt` : 등록날짜
- `activatedAt`: 활성화 날짜

- `widhdrawAt`: 탈퇴 날짜

#### 행위
- `create()`: 상세정보를 생성한다. register() 시점에 호출된다 생성자 변수는 registeredAt ( mandatory ) , address ( optional ) , profileImage( optional)
- `activate()`: 활성화 상태로 변경된 날짜를 설정한다.
- `withdraw()`: 탈퇴 상태로 변경된 날짜를 설정한다.
- `changeProfileImage()` : 증명사진을 바꾼다.
- `changeAddress()` : 주소를 변경한다.

### 회원상태 ( MemberStatus )
_Enum_
- `DEACTIVATE`: 비활성화 상태.최초 등록상태
- `ACTIVATE`: 활성화 상태
- `WITHDRAW`: 탈퇴 상태

### Email
_Value Object_
#### 속성
- `address` : 이메일 어드레스

#### 행위
- `constructor()` : valid 체크

### Password
_Value Object_
#### 속성
- `password`: plain text 

#### 행위
- `constructor()`: 제한조건 체크 ( 4 ~ 8 )

### Image
_Value Object_
#### 속성
- `url`: image 경로

#### 행위
- `constructor()`: 유효성 체크 ( image인지 체크 )
- `exist()`: image가 실제로 존재하는지

### [ 할일( Todo ) 애크리게이트 ]
### 할일(Todo)
_Aggregate Root_
#### 속성
- `id`:고유한 값
- `title`: 타이틀
- `content` : 내용
- `category`: 항목 ( TodoCategory )
- `detail`: TodoDetail 1:1
#### 행위
- `create()` : Todo 생성
  * detail.registerStartAt
  * detail.registerCompleteAt
  * detail.priority
  
- `changeContent()`: 본문내용 변경
- `changeCategory()`: 카테고리 변경


### 힐일상세 ( TodoDetail )
_Entity_
#### 속성
- `id`:고유한 값
- `registerStartAt` : 시작할 날짜
- `registerCompleteAt` : 완료될 날짜
- `startedAt` : 시작한 날짜
- `completedAt` : 완료된 날짜
- `priority` : 우선순위
- `progress` : 진행률 ( % )
- `status` : 진행상태 ( TodoStatus )
- `comment` : 지연사유

#### 행위
- `create()`: todo 상세정보 생성.default 생성자 ( registerStartAt , registerCompleteAt)
- `start()`: todo 시작
- `complete()`: todo 완료
  * progress 값도 100으로 변경된다.
- `hold()`: todo 중단
- `registerComment()`: 사유 등록
- `changePriority()`: 우순순위 변경 ( 0 - 5 )
- `changeProgress()`: 진행률 변경 ( 0 ~ 100 )
  * 100% 이면 status ( TodoStatus ) 가 자동으로 COMPLETE로 변경된다.
- `isComplete()`: todo 완료상태 유무
- `isHold()`: todo 중단상태 유무
- `isADayBeforeExpiration()` : 만료하루전 유무
- `isDelay()` : 현재 지연상태 유무

#### 규칙
- "시작"은 "START" or "COMPLETE" 상태에서는 불가능하다.
- "완료"는 "START" 상태에서만 가능하다.
- "comment 등록"은 "EXPIRE" or  "HOLD" 상태일경우에만 등록 가능하다.
- "진형률 변경"은 "START" 상태에서만 가능하다.


### 진행상태 ( TodoStatus )
_Enum_
- `READY` : 대기
- `START` : 시작
- `COMPLETE` : 완료
- `DELAY` : 지연상태 ( registerStartAt가 지나도 시작하지 않을경우 )
- `EXPIRE` : 만료상태 ( registerCompleteAt가 지나도 완료되지 않을경우 )
- `HOLD` : 중단상태

### 할일종류 ( TodoCategory )
_Enum_
- `DEVELOP`: 개발관련
- `SPORT`: 운동
- `ETC1` : ...  





## 애플리케이션 ( Application )

-- 

###  MemberRegister
_Service_

#### 행위  
- `register()` : 회원등록 ( 상태값 : DEACTIVATE )
- `activate()` : 회원의 상태값을 ACTIVATE 로 변경
- `withdraw()` : 회원을 탈퇴한다. ( )
- `updateInfo()` : 회원정보 갱신


### MemberFinder  
_Service_  

#### 행위  
- `getMember(memberId:Long)` : Member 반환
- `getMember(userId:String)` : Member 반환
- `findMembers()` : 조건에 일치하는 Member 목록 반환 

### TodoModifier
_Service_

#### 행위
- `register()`: "할일"을 등록한다.
- `complete()`: "할일"을 완료처리한다.
- `start()`: "할일"을 시작한다
- `hold()`: '할일'을 중단한다. ( 반드시 , "사유"를 작성해야 된다.)
- `changePriority()`: "할일"의 우선순위를 변경한다.
- `changeProgress()`: "할일"의 진행률을 변경한다.
- `changeContent()`: "할일"의 내용 ( content , title )을 변경한다.
